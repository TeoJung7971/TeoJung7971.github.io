### **• Word Meanings:**

- **의미(Meaning)**란 무엇인가?  
  \: 단어나 구로 표현되는 생각(idea)   
  \: 사람이 단어나 기호로 표현하고자 하는 생각(idea) 등...  
- 이를 언어학(Linguistics)에서 의미론적(Semantics) 관점으로 표현한다면?  
  \: <mark style="background-color: #e2e8f0;">signifier(symbol) ⇔ signified(idea or thing)</mark>  
  \: *지시적 의미론(denotational semantics)*  
  \: *개념이나 사물(idea or thing)이 기호(symbol)의 지시적 의미와 같음*  
    -> e.g., tree ⇔ {🌳,🌲,🌴,...}

## 2. Word2Vec Introduction

### **• Before Word Vectors:**

- **The Previously Commonest NLP Solution: WordNet**
  - WordNet: 단어 간 관계를 보여주는 고급 유의어 사전(thesaurus), \
    동의어와 상의어 등 포함  
  - 그러나 단어의 뉘앙스를 놓치는 등 의미 분석에 있어 한계 존재  
    -> *e.g., 'good'과 'proficient'는 유의어, 그러나 'good shot'은 자연스러워도 'proficient shot'은 자연스럽지 않음*  
    -> 위와 같은 예시 이외에도, 신조어를 계속해서 따라잡기 어렵고, 생성 및 적용에 있어 인력을 요구하고, 주관적이며, **단어간 유사도를 정확히 계산하지 못하는 것**과 같은 문제 존재
  - 따라, 의미를 달리 표현하는 방법이 필요  
    -> Word Vectors

### **• After Word Vectors:**

- **Representing Words as a Discrete Symbols: One-hot Vector**
  - 어휘 모음(Vocabulary)의 크기 = 벡터의 차원(Vector Dimension)  
    ![image](3.png)  
  - 오랜 기간동안 사용되어 온 방식이지만, 단어의 의미와 어떠한 자연적, 내재적 연관성이 없다는 점 등에서 문제 존재  
    -> *e.g., 호텔과 모텔은 1의 위치가 전혀 다른 두개의 벡터로 나타내어질 뿐, 그 사이에서 어떠한 연관성도 찾을 수 없음*  
    -> *수학적 관점으로는 직교하므로(orthogonal) 두 벡터의 내적이 0임*
      > $$v_1 = [1, 0, 0, 0]$$
      > $$v_2 = [0, 0, 1, 0]$$
      > 
      > $$v_1 \cdot v_2 = [1, 0, 0, 0] \cdot [0, 0, 1, 0]$$  
      > $$= 1 \times 0 + 0 \times 0 + 0 \times 1 + 0 \times 0$$  
      > $$= 0$$  
      {: .prompt-tip .small-math }  
  - 이러한 문제를 해결하기 위하여, 단어 유사성에 대한 다른 자료 구축하여 참조하는 등의 방법 사용(e.g., 쿼리 확장 기법, Query Expansion Technique)  
    -> 그러나, 여전히 자연스러운 단어 유사성 개념을 나타내기엔 부족  
    -> 벡터 자체에(내부에) 단어 유사성 개념 포함 시도
