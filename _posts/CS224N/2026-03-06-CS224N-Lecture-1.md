## 2. Word2Vec Introduction

### **• Before Word Vectors:**

- **The Previously Commonest NLP Solution: WordNet**
  - WordNet: 단어 간 관계를 보여주는 고급 유의어 사전(thesaurus), \
    동의어와 상의어 등 포함  
  - 그러나 단어의 뉘앙스를 놓치는 등 의미 분석에 있어 한계 존재  
    -> *e.g., 'good'과 'proficient'는 유의어, 그러나 'good shot'은 자연스러워도 'proficient shot'은 자연스럽지 않음*  
    -> 위와 같은 예시 이외에도, 신조어를 계속해서 따라잡기 어렵고, 생성 및 적용에 있어 인력을 요구하고, 주관적이며, **단어간 유사도를 정확히 계산하지 못하는 것**과 같은 문제 존재
  - 따라, 의미를 달리 표현하는 방법이 필요  
    -> **Word Vector**의 등장

### **• After Word Vectors:**

- **Representing Words as a Discrete Symbols: One-hot Vector**
  - 어휘 모음(Vocabulary)의 크기 = 벡터의 차원(Vector Dimension)  
    ![image](3.png)  _Cf. 위치적 표현(Localist representation)_

  - 오랜 기간동안 사용되어 온 방식이지만, 단어의 의미와 어떠한 자연적, 내재적 연관성이 없다는 점 등에서 문제 존재  
    -> *e.g., 호텔과 모텔은 1의 위치가 전혀 다른 두개의 벡터로 나타내어질 뿐, 그 사이에서 어떠한 연관성도 찾을 수 없음*  
    -> *수학적 관점으로는 직교하므로(orthogonal) 두 벡터의 내적(dot product)이 0임*
      > $$v_1 = [1, 0, 0, 0]$$  
      > $$v_2 = [0, 0, 1, 0]$$
      > 
      > $$v_1 \cdot v_2 = [1, 0, 0, 0] \cdot [0, 0, 1, 0]$$  
      > $$= 1 \times 0 + 0 \times 0 + 0 \times 1 + 0 \times 0$$  
      > $$= 0$$  
      {: .prompt-tip .small-math }  
  - 이러한 문제를 해결하기 위하여, 단어 유사성에 대한 다른 자료 구축하여 참조하는 등의 방법 사용(e.g., 쿼리 확장 기법, Query Expansion Technique)  
    \: 그러나, 여전히 자연스러운 단어 유사성 개념을 나타내기엔 부족  
    \: 벡터 자체에(내부에) 단어 유사성 개념 포함 시도  
<br/>

- **Representing Words by Their Context: Word Vector**  
  - **분포 의미론(Distributional Semantics)**  
    \: 지시적 의미론(Denotational Semantics)에 대한 대안적인 개념, 아래 J.R.Firth의 말이 이를 표현하기 위하여 자주 인용됨

    > *"You shall know a word by the company it keeps."*  
    > *"단어와 함께하는 것이 그 단어의 의미를 알린다"* 
 
    \: 즉, (대상) 단어 주변에 나타나는 단어들(문맥)이 단어의 의미에 대한 정보를 제공한다는 것
    ![image](4.png)_'banking'이라는 단어의 뜻을 알고 싶다면 해당 단어가 속한 문장과 그 주변에 나타나는 단어를 보아라_

  - **Word Vector**  
    \: Distributional Semantics 개념을 적용한 새로운 단어 표현 방식  
    \: 단어의 의미를 나타내는 더 짧고 밀집된(dense) 벡터  
    \: <mark style="background-color: #f8ff23;">단어들이 서로 관련되어 있다면, 비슷한 벡터를 가지게 되고, 해당 벡터들의 내적(dot product) 값이 큼</mark>  
    ![image](5.png)
    \: 이러한 표현 방식을 **단어 벡터(Word Vector)**라고 하며, **단어 임베딩(Word Embeddings)**이나, 신경망 단어 표현(Neural Word Representations)라고도 함  
    \: '좋은 단어 벡터'= 단어의 의미를 잘 나타내는 벡터 
      > **Embeddings?**  
      > • 의미: *어떤 위상(位相) 공간에서 다른 위상 공간으로의 동상 사상(同相寫像)* \\
      > • 즉, 자연어 처리에서는 자연어를 벡터로 바꾸어 벡터 공간(vector space)에 넣은 결과 또는 넣는 과정을 의미  
      {: .prompt-tip .small-math }  
  
  - **Word Vector and Vector Space**
    ![image](6_.png)_단어 벡터가 임베딩된 벡터 공간 시간화 예시(t-SNE 차원 축소 기법 사용) \
      비슷한 의미, 비슷한 문법적 성질을 지닌 단어들이 서로 가깝게 모여 있는 모습을 볼 수 있음_  

### **• Word2Vec Overview:**

- **What is Word2Vec?**
  - [📖Original Paper: Efficient Estimation of Word Representations in Vector Space](https://arxiv.org/pdf/1301.3781)
  - 구글의 Mikolov와 그의 동료들이 발표, 이전에도 단어 벡터 학습 방법에 대한 연구는 존재했으나, Word2Vec은 다른 방법들보다 매우 간단하고 빨라 주목받게 됨  
<br/>

- **Idea of Word2Vec**
  - Large Corpus(Body) of Text, 즉, 단어의 긴 리스트를 지니고 있음
  - 고정된 어휘 목록(Vocabulary)에 있는 각 단어들은 하나의 벡터로 표현됨
  - Text 내의 <mark>각 <em>위치</em> $t$</mark>를 살피어 봄 \\
    이때, 텍스트 내의 *위치에는* <mark>중심 단어인 $c$와 문맥(바깥) 단어인 $o$가 존재</mark>
  - **$c$가 주어질 때 $o$가 등장할 확률을 계산**하기 위하여 $c$와 $o$의 **단어 벡터 간 유사도를 이용** (혹은 그 반대로)
  - 이때, (등장) **확률을 최대로 하는 방향**으로 **단어 벡터를 계속하여 조정**  
<br/>

- **Overview**
  ![image](7.png) _window size $m$ = 2, center word $c$ = into인 강의 예제_  
  - 단어가 나타나는 위치를 가리키는 **<mark style="background-color: #f8ff23;">단어 유형(word type)</mark>**에 대한 벡터 생성
  - 단, 단어 유형은 개별 단어 토큰(예: *'problems'*라는 단어 객체)과는 구별됨  
  -> e.g., 위의 이미지에서 *'turning'*이라는 단어가 *'into'*라는 단어 앞에 (혹은 가까운 위치)에 등장할 확률을 계산, 최종적으로 해당 확률 추정치를 가장 정확하게 만드는 것이 목표  
  -> e.g., *banking* 뒤에 *crises*가 이어질 확률은 크지만, *banking* 뒤에 *skillet과* 같은 (관련성이 떨어지는) 단어가 올 확률은 적게끔 만들기  
<br/>

## 3. Word2Vec Objective Function Gradients

### **• Objective function:**

- **Notation**
  - 각 position $t = 1, ..., T$
  - 고정된 window 크기 $m$ (강의 내 예제에서는 $m=2$) 
  - 전체 파라미터 $\theta$ (강의 내에서는 연결된 전체 word vectors를 의미함)  
<br/>

- **Formula: Likelihood(가능도)**  
  
  $$L(\theta) = \prod_{t=1}^{T} \prod_{\substack{-m \le j \le m \\ j \ne 0}} P(w_{t+j} \mid w_t ; \theta)$$


- **Formula: <mark>Average Negative Log</mark> Likelihood(가능도)**  
  
  $$J(\theta) = -\frac{1}{T} \log L(\theta) = -\frac{1}{T} \sum_{t=1}^{T} \sum_{\substack{-m \le j \le m \\ j \ne 0}} \log P(w_{t+j} \mid w_t ; \theta)$$  

  - **Average:** $\frac{1}{T}$
  - **Negative:** $-$  
    -> 최대화(maximization) 문제에서 최소화(minimization) 문제로 변형
  - **Logarithm:** $\log$  
    -> 곱연산($\Pi$)에서 합연산($\Sigma$)으로 변경. 연산량 감축

### **• Calculate Probability:**

- **How to Calculate $$P(w_{t+j} \mid w_t ; \theta)$$?**

  $$P(o|c) = \frac{\exp(u_o^T v_c)}{\sum_{w \in V} \exp(u_w^T v_c)}$$

  - **내적(dot product):** $$u_o·v_c$$  
    -> 유사도(similarity) 계산
  - **지수화(exponentiation):** $$exp()$$  
    -> 내적 값의 범위는 실수 전체 범위(unbounded)이므로 음수값이 나올 수 있음. 이를 방지하지 위하여 지수 함수를 취함  
    -> e.g., ${e}^{x}$, 이때 $x$는 내적 연산 결과
  - **정규화(*(statistical)* Normalization):** $$\sum_{w \in V}$$  
    -> 전체 합이 1이 되는 확률 분포 형태로 정규화함  

  > **Softmax Function**  
  > • 수식: $$\text{softmax}(x_i) = \frac{\exp(x_i)}{\sum_{j=1}^{n} \exp(x_j)} = p_i$$  
  > • 효과: $$\mathbb{R}^n \rightarrow (0,1)^n$$  
  > -> 모든 실수의 집합이 정의역일 때, 그 치역은 0과 1 사이의 값으로 확률 분포를 나타내게 됨  
  > -> 이때 $$exp()$$ 연산의 특성 상, 소프트맥스 함수를 거친 값이 0이나 1에 해당할 수는 없음, 반드시 0 초과거나 1 미만이게 됨  
  {: .prompt-tip .small-math } 

### **• Train the Model:**  

- **How to train the model?**

  - 핵심은 <mark>Loss값을 최소화</mark>할 수 있도록 파라미터($$\theta$$)를 점진적으로 조정해나가는 것  
  ![image](8.png)_해당 이미지의 중앙에는 $$\theta$$에 대한 예시가, \
  우측에는 2차원 상 그래디언트와 최적화(minima 탐색)에 대한 예시가 있음_  
  - $$\theta$$는 전체 모델 매개변수를 가리키며, 해당 경우에는 길고 단일한 벡터
    - 앞서 설명한 word type 별로(context, center) 배정된 각 단어의 두 word vectors가 연결된(concatenation) 형태  
    -> e.g., 각 단어 벡터의 크기 $$d = 100$$, 전체 어휘의 수 $$v = 400,000$$라고 하면, $$\theta$$의 크기는 $$100*2*400,000=80,000,000$$(8천만)
  - 즉, $$\theta$$가 나타내는 모든 매개변수의 그래디언트(gradient: 다차원상의 기울기)를 계산한 뒤, 이를 이용하여 모든 파라미터를 최적화하는 과정을 거치게 됨
